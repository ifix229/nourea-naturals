The Netlify deploy errored, with the following guidance provided:

- Relevant error lines
  - [line 21](#L21) and [line 22](#L22) show the immediate failure: npm failed with EUNSUPPORTEDPROTOCOL and "Unsupported URL Type "catalog:"".

- Error type and cause
  - Type: npm install failure (EUNSUPPORTEDPROTOCOL).
  - Cause: One or more dependency references use the unsupported "catalog:" protocol. npm does not recognize "catalog:"; that protocol is produced/used by other package managers (e.g., pnpm/Yarn metadata or a lockfile format) or by a malformed dependency entry. npm therefore aborts the install.

- Concrete solution (steps)
  1. Reproduce / locate the offending reference locally:
     - From your repo root run:
     ```bash
     # find any occurrence of the catalog: protocol
     grep -R "catalog:" -n || true
     ```
     - Inspect any results (package.json, package-lock.json, pnpm-lock.yaml, yarn.lock, or other files).

  2. Fix depending on what you find:
     - If the "catalog:" entry is in package.json (dependencies/devDependencies):
       - Replace the "catalog:..." spec with a standard spec npm understands (version, tarball URL, git URL, or registry name). Example:
         - Bad: "some-package": "catalog:some-ref"
         - Good: "some-package": "^1.2.3"  or "some-package": "git+https://github.com/owner/repo.git"
       - Commit package.json and push.
     - If the "catalog:" entry is only present in a lockfile generated by another package manager (e.g., pnpm-lock.yaml or yarn 3 metadata) and your project is intended to use pnpm/yarn:
       - Ensure the lockfile for that package manager is committed (e.g., commit pnpm-lock.yaml).
       - Instruct Netlify to use that package manager so it doesn't run npm:
         - Option A (preferred): Commit pnpm-lock.yaml and Netlify will auto-detect pnpm and use pnpm install. If it doesn't detect, set the install command in Netlify to:
           - Install command (Netlify site UI or netlify.toml):
           ```
           pnpm install
           ```
           - Or change the build command to:
           ```
           pnpm install && npm run build
           ```
         - Option B: Explicitly run pnpm in your build by editing the Netlify Build Settings "Build command" to: `pnpm install && npm run build`, and ensure pnpm-lock.yaml is committed.
     - If the "catalog:" entry is in package-lock.json produced by a different environment:
       - Delete package-lock.json, regenerate with your intended package manager (npm i or pnpm i), commit the correct lockfile, and push.

  3. Rebuild on Netlify.

- Verification tips
  - Run npm install locally to reproduce the error if npm is your intended installer. If you intend to use pnpm, run pnpm install locally to confirm success.
  - If you switch to using pnpm on Netlify, make sure pnpm-lock.yaml is committed so Netlify auto-detects it, or set the install command explicitly as above.

Notes
- This error is not a Node version mismatch, so no Node docs link is required.
